Beschreibung

Ein Befehl (XYZ.class extends ACmd) dient dazu, einen Befehl zur Ausführung zu bringen, indem er eine spezifische Anforderung über die Methode public final void command() annimmt, aufbereitet und im Receiver die entsprechende Methode aufruft.

Die Instanzierung erfolgt über den Konstruktor Order(name, manuell). Erster Parameter ist der Eingabecode des Befehls, der zweite Parameter bestimmt, ob der Befehl manuell aufrufbar ist oder nur von einem Automaten bedient wird.

Die Methoden public String testParameter(String parameter), public int maxParameter(String parameter) und public int minParameter(String parameter) dienen zur Parametersteuerung.

Die Informationskette läuft von:

  1. Einsprung in public final void command()
  2. Parameterverarbeitung in public static final void Command.parameterCheck(this, false)
  3. Aufruf des generellen Calls public void call(String parameter, String... value)
  4. Daraus erfolgt der Aufruf des parameterspezifischen Calls im Enumeration c: public void call(ICmd cmd, String... value). Ggfs. können hier auch kleinere Datenverarbeitungen stattfinden.
  5. Aufruf der Receiverfunktion cmd.getReceiver().methode()

Ist kein Parameter vorhanden, kann schon in public final void command() der Receiver aufgerufen werden.

Die Parameter werden in folgender Form eingeben:

  • show: Wird angezeigt.
  • buttontext: Beschriftung des Buttons im SWT
  • tooltiptext: Hinweistext im SWT
  • parameterMin: Mindestanzahl der Parameter
  • parameterMax: Maximalanzahl der Parameter
. Der Aufruf erfolgt im Parameter-Enummeration c mit paramter(show,buttontext,tooltiptext,paramtermin, paramtermax) Einige Parameter müssen implementiert sein:
  • help, ?, h
      die Hilfe zu diesem Befehl
  • tst
      der Schalter, um den Befehl nur zu testen, aber nicht auszuführen
